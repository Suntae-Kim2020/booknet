# 소셜 로그인 구현 오류 & 해결 정리

## 1. 앱이 흰 화면만 나오고 멈춤

**원인**: Flutter **debug 모드**로 빌드하면 앱이 개발 도구(VM Service)에 연결을 시도합니다. 연결이 안 되면 앱이 멈춰서 흰 화면만 보입니다.

**해결**: `flutter build ios --release`로 **release 모드**로 빌드하면 개발 도구 연결 없이 독립 실행됩니다.

---

## 2. 카카오 로그인 — "iOS bundleId validation failed"

**원인**: 앱의 번들 ID(`io.booknet.booknet`)가 카카오 개발자 콘솔에 등록되지 않았거나, `.env`의 앱 키가 콘솔의 **네이티브 앱 키**와 달랐음.

**해결**:
- 카카오 개발자 콘솔 → **앱** → **플랫폼 키** → iOS 번들 ID 등록
- `.env`의 `KAKAO_NATIVE_APP_KEY`를 콘솔의 네이티브 앱 키와 일치시킴
- `Info.plist`에 URL scheme `kakao{네이티브앱키}` 등록

---

## 3. 카카오 로그인 — 로그인 후 앱으로 안 돌아옴

**원인**: 원래 코드는 카카오SDK → Firebase Custom Token → Supabase라는 복잡한 경로를 사용했고, 중간에 필요한 Edge Function이 없었음 (404 에러).

**해결**: 복잡한 경로 대신 **Supabase가 기본 지원하는 카카오 OAuth**를 사용.
- Supabase 대시보드 → Authentication → Providers → Kakao 활성화
- `signInWithOAuth(OAuthProvider.kakao)` 한 줄로 로그인 처리
- Supabase에서 Redirect URL 설정, 카카오 콘솔에서 Redirect URI 등록

---

## 4. 카카오 로그인 — 인앱 브라우저에서 "계속하기" 눌러도 반응 없음

**원인**: `LaunchMode.inAppBrowserView`로 열었을 때 콜백 리다이렉트가 제대로 작동하지 않음.

**해결**: `LaunchMode.externalApplication`으로 변경. 외부 Safari에서 로그인하면 딥링크로 앱에 정상 복귀.

---

## 5. 네이버 로그인 — "서비스 설정에 오류가 있어 로그인할 수 없습니다"

**원인**: 네이버 개발자 센터에서 **Mobile 웹** 환경을 추가하지 않아서 **Callback URL**을 등록할 곳이 없었음. iOS 환경만 추가하면 Callback URL 필드가 나오지 않음.

**해결**:
- 네이버 개발자 센터 → API 설정 → **환경 추가** → **Mobile 웹** 선택
- 서비스 URL과 **Callback URL** 입력
- 서비스 URL 입력 후 반드시 **+ 버튼**을 눌러야 실제 등록됨

---

## 6. 네이버 로그인 — "Token has expired or is invalid"

**원인**: Edge Function에서 세션을 미리 만들어서 URL로 전달하면, 리다이렉트 과정에서 토큰이 만료됨.

**해결**: 2단계로 분리:
1. **auth-naver-callback** (Edge Function): 네이버 코드를 받아 access_token으로 교환 → 앱으로 전달만 함
2. **앱에서 직접** auth-naver Edge Function을 POST 호출 → 세션을 즉시 생성하여 만료 방지

---

## 핵심 교훈

| 항목 | 내용 |
|------|------|
| **빌드 모드** | iOS 실기기 테스트는 반드시 `--release` 모드 |
| **앱 키 확인** | `.env`의 키와 개발자 콘솔의 키가 일치하는지 항상 확인 |
| **URL scheme** | `Info.plist`에 각 서비스별 URL scheme 등록 필수 |
| **Callback URL** | 개발자 콘솔에서 정확한 Callback URL을 등록하고, **+ 버튼** 클릭 후 **저장** |
| **환경 추가** | 네이버는 Mobile 웹 환경을 추가해야 Callback URL 필드가 나옴 |
| **토큰 전달** | 세션 토큰을 URL로 전달하면 만료됨 → 앱에서 직접 API 호출로 세션 생성 |
| **간단한 구조** | Supabase 기본 지원 OAuth(카카오, 구글)를 우선 사용. 미지원(네이버)만 Edge Function 사용 |
